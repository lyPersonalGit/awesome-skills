# 重构建议检查清单

在提供重构建议时，检查以下重构点：

## 1. 提炼函数
- 超过30行的代码段是否可以拆分
- 一段代码做了清晰可分的两件事是否可以拆分

## 2. 提炼变量
- 复杂的条件表达式是否可以提炼命名变量
- 魔法值是否可以提炼常量

## 3. 变量重命名
- 变量名是否模糊不清晰
- 变量名是否误导
- 临时变量生命周期是否过长可以改名

## 4. 改变函数声明
- 函数名是否和实际业务不符
- 参数顺序是否合理
- 参数是否可以合并

## 5. 优化条件表达式
- 嵌套条件是否可以用卫语句减少嵌套
- `&&` 超过三个是否可以拆分提炼变量
- 颠倒条件判断是否让主干路径更清晰

## 6. 引入参数对象
- 参数列表是否有多个参数总是一起出现
- 是否可以合并为一个参数对象

## 7. 分解条件表达式
- 复杂的if-else是否可以拆分多个函数

## 8. 替换魔法值
- 魔法数字是否替换为命名常量
