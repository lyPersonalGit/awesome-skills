# Generate Java Tech Doc

本技能用于生成标准化的Java技术文档，支持从Java项目文件自动生成，也支持改造已有的技术文档。

## 文档结构

生成的技术文档必须包含以下章节，严格按照顺序排列，并为所有标题添加序号：

1. **1 功能概述** - 实现了哪些功能，解决了哪些问题，有哪些核心特性
2. **2 快速开始** - 添加依赖、配置文件、配置类等步骤
3. **3 实现方案** - 贴入主要功能的完整代码实现
4. **4 使用示例** - 具体使用场景和代码示例
5. **5 常见问题** - 列出常见问题和解决方案
6. **6 最佳实践** - 推荐的使用方式和注意事项
7. **7 参考资料** - 相关链接和参考文献

### 标题编号规则

- 一级标题（##）使用阿拉伯数字编号：`## 1 功能概述`、`## 2 快速开始`
- 二级标题（###）使用点号分层编号：`### 2.1 添加依赖`、`### 2.2 配置文件`
- 三级标题（####）继续分层编号：`#### 2.1.1 Maven`、`#### 2.1.2 Gradle`

## 工作流

### 生成新文档

1. **分析项目**：使用glob和grep工具扫描Java项目文件，理解项目结构和功能
2. **提取信息**：从代码中提取功能特性、API信息、依赖配置等
3. **生成文档**：按照标准模板生成完整带序号的文档
4. **验证完整性**：检查所有必填章节是否都已填充，标题编号是否正确

### 修改已有文档

1. **读取现有文档**：获取当前文档完整内容
2. **对比代码分析**：对比代码更新，找出需要修改的部分
3. **保留知识内容**：**尽量不要减少文档已包含的知识内容和代码内容**
4. **修正技术谬误**：对已发现的明显技术错误必须指正并修改
5. **重新编号整理**：为所有标题添加标准序号，整理格式使其符合规范
6. **更新变更内容**：更新因代码变更而需要修改的文档内容

## 代码提取规则

1. **主要类完整显示**：核心功能类需要展示完整源代码
2. **关键方法重点说明**：对复杂方法添加注释说明
3. **保留包结构**：保持正确的package和import声明
4. **移除测试代码**：除非文档专门针对测试

## 修改已有文档规则

- **保留内容优先**：尽量不要减少文档已包含的知识内容和代码内容
- **修正技术错误**：如果发现明显的技术谬误，必须指正并修改
- **统一编号格式**：为所有标题重新添加标准序号，保持格式一致
- **结构标准化**：将原有内容整合到标准文档结构中，不丢弃有价值信息

## 使用场景

- 新建Java项目需要生成项目文档
- 为现有Java库补充技术文档
- 改造旧的Java项目文档使其标准化，添加标题序号
- 根据源码自动生成API使用文档
- 更新已有的技术文档以匹配最新代码变更
